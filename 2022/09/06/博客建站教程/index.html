博客建站教程

我基本没有前端方面的知识，搭建博客都是按着网络教程进行，很多概念也并不理解，所以这里只记录一下我在搭建过程中的一些经验。

技术栈

  • Git
  • github
  • github pages
  • Hexo
  • GitHub Actions
  • html、css、web等前端的一些基础知识

根据这些关键词可以自行搜索相关资料。

在几个月前我已经在腾讯云上购买好了轻量云服务器和个人域名，并且用wordpress框架+sakura主题初步配置好了博客网站。而后舍弃了，原因如下：

  1. 我了解到了Github Pages，比起wordpress，它对Markdown文件的支持更加优秀，而且还能够和Github相结合
  2. 云服务器我还是想用来装Linux系统折腾些别的项目，而非是单纯用来挂博客

部署Hexo

参考：【2021最新版】保姆级Hexo+github搭建个人博客_哔哩哔哩_bilibili

虽然官方文档最权威，但是当前我没有足够的知识去理解官方文档里面的说明，所以最好跟着时间较近的视频类教程部署。

下面只大致说下我在跟着这个教程的时候遇到的坑

当然这部分也可以直接看官方文档

  1. 安装node.js：注意官网文档里面有列举”Node.js 版本限制“，不推荐版本太老，我装的是node-v16.17.0-x64，在2022年9月5日属于当前稳定版本。安装教程很多，自己找。重点：最后配置node_global和node_cache文件夹。其实环境变量配置熟练了也很简单。然后有个坑，后续在cmd中用npm指令安装测试失败，原因是权限不够，cmd要用管理员权限打开。
  2. 安装git：很早就安装好了，不记录
  3. github仓库：完成，坑：git push origin master与git push origin main，我的主支是main，很多时间较老的教程，主支是master，注意区分。(这是因为Github更新了默认主支)
  4. hexo本地部署：hexo init需要在空文件夹中进行，同时视频教程里面使用的是git Bash，个人推荐还是使用管理员模式打开的cmd来完成，然后我直接使用魔法下载，所以不考虑换源。
  5. 我hexo init完，后续的指令全是在hexo部署的文件夹路径下完成的

后面我再参考了：GitHub Pages + Hexo搭建个人博客网站，史上最全教程_淡淡的说非的博客-CSDN博客，文字版教程，Fluid主题比较喜欢

然后推荐下这个教程：hexo和github搭建个人博客快速教程，推荐的原因：小姐姐声音甜美。

这里我提一下，我见到有些教程是说随意命名一个仓库，然后设置成github pages，但是还有一些教程说仓库的命名一定要是username.github.io，我是按照后面的方式进行，没有详细了解造成这种差异的原因，个人还是推荐按照后者的方式进行。

我的本地博客文件夹如上图，文件夹的名称是aqstoner.github.io就是仓库名，然后和github的仓库之间联动有两种方式：

  1. token
  2. ssh

有的教程用方式1，有的用方式2，使用其中一种即可，我采取的是后者，至于两个方式的详细步骤和相关知识，自行搜索解决。

自定义主题

虽说是自定义主题，也就是看看Fluid的官网文档，然后按照它的讲解改下配置文件而已，毕竟我也不怎么会css和html

Hexo写博客

关于这个主要看Hexo的官方文档，最基本的就是掌握这几个指令

  1. hexo g
  2. hexo s
  3. hexo d
  4. hexo g -d
  5. hexo clean
  6. hexo new post "title"
  7. hexo new page "title"
